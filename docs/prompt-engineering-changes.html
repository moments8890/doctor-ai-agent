1. Routing loaded KB unnecessarily ChatGPTClaudeCodex

Routing LLM only classifies intent. Loading doctor KB adds noise, cost, and can bias intent classification. Comment said "no knowledge" but config said True.

+ ROUTING_LAYERS = LayerConfig(intent="routing", load_knowledge=False)
2. Pattern 2 trust inversion Codex
src/agent/prompt_composer.py:106-127
Conversation mode put doctor-authored KB into system message, giving user-generated content system-level authority. base.md says KB is subordinate to intent rules, but message placement contradicted this.
Before (Pattern 2):
- system_msg += doctor_knowledge # KB in system = high trust
- system_msg += patient_context
 
After (Pattern 2):
+ system_msg += patient_context # factual DB data OK in system
+ user_msg = <doctor_knowledge>... # KB in user msg = correct trust
