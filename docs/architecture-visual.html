What Is This?

A personal AI assistant for doctors managing private patients outside hospitals. Doctors dictate medical records, get AI-powered differential diagnoses, manage follow-up tasks, and communicate with patients. NOT an EMR — a lightweight clinical productivity tool.

Three channels: Web (React SPA), WeChat/WeCom, Patient portal. Backend: FastAPI with Plan-and-Act agent pipeline. LLM: Qwen3 via OpenAI-compatible API with instructor for structured output.

Process Model

How the app runs: single async process, background workers, scheduled jobs, and fire-and-forget tasks. Everything is asyncio — no threads, no multiprocessing, no Celery.

Always-Running Background Workers

Started in lifespan, run until shutdown. Both use asyncio.Queue for backpressure.

WorkerQueue SizeBehaviorLocation
Span Disk Writer 5,000 Writes trace spans to JSONL; prunes every 200 writes; falls back to sync write when queue full src/infra/observability/_storage.py
Audit Log Drain 2,000 Batches audit entries (100/batch), flushes to DB every 5s; 3 retries on DB failure; graceful drain on shutdown src/infra/observability/audit.py

Scheduled Jobs (APScheduler)

Configurable via TASK_SCHEDULER_MODE (interval or cron). Default: interval. Multi-instance safe via DB lease (TTL 90s, owner = hostname:pid).

JobScheduleWhat it doesLocation
Task Notifications Every 1 min Polls doctor_tasks for due items, sends WeChat notifications, transitions pending → notified src/domain/tasks/task_crud.py
Chat Log Cleanup Daily 04:30 Hard-deletes chat log entries older than 365 days src/startup/scheduler.py
Audit Retention Monthly (1st, 03:00) Deletes audit log entries older than 7 years src/startup/scheduler.py
Turn Log Pruning Daily 05:30 Prunes conversation turn logs src/startup/scheduler.py

Fire-and-Forget Tasks

Per-request async tasks via safe_create_task() (src/utils/log.py). Wraps asyncio.create_task() with error logging — exceptions are caught and logged, never crash the request.

TriggerBackground TaskWhat happensLocation
WeChat doctor message _handle_intent_bg() Routes through agent pipeline, sends reply via WX API (4.5s timeout) src/channels/wechat/router.py
WeChat patient message _handle_patient_bg() Emergency keyword check → triage → agent → auto-reply src/channels/wechat/router.py
Image upload (WeChat) handle_image_bg() Download media → vision LLM OCR → send result to doctor src/channels/wechat/flows.py
PDF/Word upload handle_pdf_file_bg() Extract text → optional LLM cleanup → send result src/channels/wechat/flows.py
Diagnosis confirmed run_diagnosis() LLM generates differentials → saves to ai_suggestions → logs KB citations; follow-up tasks are created only after doctor review finalize src/domain/diagnosis_pipeline.py
Triage escalation handle_escalation() LLM summarizes patient concern → creates review task for doctor → sends notification src/domain/patient_lifecycle/triage_handlers.py

Concurrency Summary

PatternMechanismUsed by
Request handling async def + await All HTTP endpoints
Background workers asyncio.create_task() in lifespan Span writer, audit drain
Scheduled jobs APScheduler AsyncIOScheduler Task notifications, cleanup
Per-request background safe_create_task() (fire-and-forget) WeChat handlers, document ingestion, triage
Multi-instance safety DB lease (hostname:pid, 90s TTL) Scheduler job dedup

No threads, no multiprocessing, no Celery, no Redis queues. Everything runs on a single asyncio event loop. Production uses uvicorn --workers N for process-level parallelism.
