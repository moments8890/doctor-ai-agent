What Is This?

A personal AI assistant for doctors managing private patients outside hospitals. Doctors dictate medical records, get AI-powered differential diagnoses, manage follow-up tasks, and communicate with patients. NOT an EMR — a lightweight clinical productivity tool.

Three channels: Web (React SPA), WeChat/WeCom, Patient portal. Backend: FastAPI with Plan-and-Act agent pipeline. LLM: Qwen3 via OpenAI-compatible API with instructor for structured output.

Composition Patterns

Pattern 1: Single-Turn

Used by query_record, query_task, query_patient, create_task, general, daily_summary.

user: <knowledge>L4</knowledge>
<patient_context>L5</patient_context>
<user_message>L6</user_message>

Pattern 2: Conversation

Used by create_record (interview mode).

system: L1 + L2 + L3 + L4 + L5
history: prior turns...
user: L6 (plain text)

Medical record state transitions — src/db/models/records.py RecordStatus enum.

Shortcut path (all fields present at confirm):

interview_active

When the interview collects all required fields and the doctor confirms, the record skips pending_review and transitions directly to completed.
