Animation Decision Logic

Scenario Mechanism Animation Why
Tab ↔ Tab (doctor: 4 mobile / 5 desktop) SectionContent renders inline INSTANT wx.switchTab — tabs are peers, no animation
Tab ↔ Tab (patient: 4 tabs) Conditional render (no Fade) INSTANT wx.switchTab — tabs are peers, no animation
List → Subpage (push) PageSkeleton → SlideOverlay, direction="forward" SLIDE wx.navigateTo — ~300ms, ease [0.32,0.72,0,1], edge shadow, -24% parallax exit
Subpage → List (← arrow tap) markIntentionalBack() + navigate(-1) SLIDE wx.navigateBack — intentional back, animate pop
Subpage → List (iOS swipe / browser back) Browser POP, no intentional flag INSTANT (iOS) iOS/WeChat WebView renders own swipe visual — no double play
Subpage → List (Android / non-iOS back) Browser POP, non-iOS detected SLIDE No native visual on Android WeChat WebView — we provide it
Subpage → deeper subpage (knowledge → detail) subpageKey changes, direction="forward" SLIDE wx.navigateTo — deeper navigation, standard push
Review record overlay SlideOverlay in DoctorPage, z-index 5 SLIDE Sits above all tabs + bottom nav (like page-container position=right)
Deep link / first render direction="none", AnimatePresence initial=false INSTANT No animation on cold start / deep link
prefers-reduced-motion useReducedMotion() → direction="none" INSTANT Respect accessibility preference

Component Stack (Mobile)

Rendering layers from outermost to innermost on mobile.

MobileFrame
  └─ RequireAuth
      └─ ApiProvider
          └─ DoctorPage — owns activeSection, bottom nav, review overlay
              ├─ SectionContent — instant tab switch, no animation
              │   └─ PatientsPage / SettingsPage / TaskPage / ...
              │       └─ PageSkeleton — list + mobileView
              │           ├─ listPane — always visible
              │           └─ SlideOverlay — animates mobileView (subpages)
              │               └─ AnimatePresence — keyed by subpageKey
              │                   └─ motion.div — push/pop tween
              │
              ├─ SlideOverlay (review) — z=5, above everything
              │   └─ ReviewPage
              │
              └─ MobileBottomNav — hidden when isSubpage=true
