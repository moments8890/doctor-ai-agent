## Defense stack — what protects each surface today

### Authentication path

Code: `src/infra/auth/unified.py`, `src/channels/web/auth/unified.py`.

| Layer | Mechanism | Where |
|---|---|---|
| Hashing | PBKDF2-SHA256, 600k iters, per-row salt | `utils/hashing.py` (reused from access-code path) |
| Lockout | 5 failures → 7-day lock | `_atomic_record_login_failure`; tunable via `LOGIN_FAIL_THRESHOLD` / `LOGIN_LOCK_SECONDS` |
| Atomic counter | SQL `UPDATE ... CASE` (race-free) | same |
| DOS prevention | counter mutates only when (nickname, role) narrows to ONE row; cross-role ambiguity never penalizes either account | `login()` failure path |
| Selector required | `login_with_role(role=patient)` requires `doctor_id` or `patient_id` (HTTP 400) | `login_with_role()` |
| Timing parity | dummy PBKDF2 verify on missing-user AND locked-account paths | `_dummy_hash()` |
| Status code | uniform HTTP 401 for every failure mode | `login()` |
| IP rate limit | 30/min/IP on `/unified/login` and `/unified/login-role` | `enforce_ip_rate_limit` in `infra/auth/rate_limit.py` |
| Token TTL | 7 days | `_TOKEN_TTL` env `UNIFIED_TOKEN_TTL` |
| Revocation | `pcv` JWT claim, re-checked vs DB on every authenticated request | `_enforce_passcode_version()` |
| Token-for-deleted-user | rejected with 401 "Token revoked" | same (missing row → 401) |
| Right-to-be-forgotten | `POST /api/auth/unified/forget-me` requires current passcode + cascades via FK | `forget_me()`; migration `a4b5c6d7e8f9` |
| QR token pcv | loads doctor/patient `passcode_version` before `issue_token()` | `generate_qr_token()` |

### LLM data flow

Code: `src/infra/llm/egress.py`, `src/agent/prompt_safety.py`, `src/agent/prompt_composer.py`, `src/domain/records/vision_import.py`, `src/domain/knowledge/knowledge_ingest.py`.

| Tier | Providers | Gate |
|---|---|---|
| LOCAL | ollama, local, lmstudio | always allowed |
| IN_CHINA | tencent_lkeap, siliconflow, dashscope, deepseek | requires `PHI_CLOUD_EGRESS_ALLOWED=true` |
| CROSS_BORDER | groq, openai, gemini, anthropic, cerebras, sambanova, openrouter, xai | requires BOTH `PHI_CLOUD_EGRESS_ALLOWED` AND `PHI_CROSS_BORDER_ALLOWED` |
| UNKNOWN | anything else | fail-safe to CROSS_BORDER (most restrictive) |

Prompt-injection containment: `wrap_untrusted(tag, content)` HTML-escapes
`<`, `>`, `&` so embedded close tags can't break out. Applied at every
user-content boundary in `prompt_composer.py` plus the OCR /
PDF-text substitutions in `vision_import._extract_fields` and
`knowledge_ingest._llm_process_knowledge`.

Output moderation: `cited_knowledge_ids` filter strips hallucinated
citations server-side; `grounded` boolean surfaced on each suggestion;
`audit_log` row written when a doctor confirms an ungrounded suggestion
(resource_type=`suggestion_confirmed_ungrounded`).

## How to verify (runbook)

### Auth

```bash
# Healthcheck
curl -sS https://api.doctoragentai.cn/healthz   # → "ok"

# Real account works
curl -sS -X POST https://api.doctoragentai.cn/api/auth/unified/login \
  -H "Content-Type: application/json" \
  -d '{"nickname":"test","passcode":"123456","role":"doctor"}'   # → 200, JWT issued

# Uniform 401 across failure modes
for body in \
  '{"nickname":"missing","passcode":"x","role":"doctor"}' \
  '{"nickname":"test","passcode":"wrong","role":"doctor"}'; do
  curl -sS -o /dev/null -w "%{http_code}\n" \
    -X POST https://api.doctoragentai.cn/api/auth/unified/login \
    -H "Content-Type: application/json" -d "$body"
done
# Expect both → 401

# Patient login without selector → 400
curl -sS https://api.doctoragentai.cn/api/auth/unified/login-role \
  -X POST -H "Content-Type: application/json" \
  -d '{"nickname":"x","passcode":"y","role":"patient"}'
# → {"detail":"patient login requires doctor_id or patient_id"}
```
