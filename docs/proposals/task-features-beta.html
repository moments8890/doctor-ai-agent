1. Snooze / Reschedule

DO NOW BOTH AGREE

Problem: Overdue tasks pile up. Doctor can't act today but has no way to defer. Result: a "wall of red" they learn to ignore.

Risk if skipped: Doctors abandon the task tab within weeks.

Proposed UX

Tap the overdue label (e.g. "逾期3天") to open a bottom sheet with quick-pick dates. No swipe gestures — WeChat users don't expect them.

Implementation

Tap overdue label → SheetDialog with 3 presets + date picker → PATCH /api/tasks/{id}/due. No new API, no new DB field. Existing endpoint already supports this.

2. AI Provenance

DO NOW BOTH AGREE

Problem: Doctor sees AI-generated task but can't verify why it was created. One unexplained task poisons trust in ALL AI tasks.

Risk if skipped: Trust collapse is non-linear. One bad unexplained task → doctor ignores all AI tasks.

Proposed UX

Show a source card on the task detail page. One tap to see the original record. No inline expansion, no citation UI — just: label + link.

Implementation

3. Overdue-only WeChat Push

DO NOW BOTH AGREE

Problem: Pushing ALL due tasks gets the miniprogram muted within days. WeChat doesn't re-prompt for notification permission — once muted, permanently muted.

Risk if skipped: You get one shot at notification credibility. Waste it on noise and you lose the push channel forever.

Proposed change

One daily digest at 08:30 for overdue items only. No per-task pushes. No "due today" pushes — the in-app badge handles that.

Implementation

5. Task Dedup

DEFER BOTH AGREE

Why: At 10-50 patients, duplicates are a mild annoyance. Fuzzy dedup is deceptively hard — "复查血压" vs "监测血压变化" are different clinical actions. A false merge is worse than a duplicate reminder.

When to revisit: When record volume per patient increases and you have real examples to define "same task" accurately.

6. Recurring Tasks

SKIP BOTH AGREE

Why: The AI pipeline already handles this organically. Doctor sees patient again → new record → AI generates next follow-up. Recurrence adds schema complexity (pause, skip, end date, interval changes) for a workflow the system already covers.

If ever needed: Simplest model = auto-clone completed task N days later. No calendar rules.
