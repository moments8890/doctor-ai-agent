# 06｜备案与合规执行清单

## 目标

在上线周内完成备案提交，并形成医疗数据合规最小闭环。

## 子清单

1. 备案动作
- [ ] 域名实名信息与主体信息一致
- [ ] 腾讯云备案前置检查通过
- [ ] 提交 ICP 备案材料并保留受理编号
- [ ] 记录审核状态与预计反馈时间

2. 数据合规基线
- [ ] 列出个人信息字段清单（患者标识、联系方式、病历）
- [ ] 设置最小化采集原则（只采业务必需字段）
- [ ] 明确数据用途、保存期限、访问角色
- [ ] 关闭非必要跨境传输

3. 安全控制
- [ ] 管理员账号启用 MFA
- [ ] 生产环境密钥统一托管，禁止硬编码
- [ ] 数据库/对象存储开启访问审计
- [ ] 关键操作保留日志（读取、导出、删除）

4. 制度文档（最小集）
- [ ] 用户隐私政策（含医疗场景说明）
- [ ] 数据安全管理制度（内部）
- [ ] 数据访问审批流程
- [ ] 安全事件上报流程

## 输出物

1. 备案受理信息截图/编号
2. 《数据资产与权限矩阵》
3. 《最小合规基线》文档

## 验收

- [ ] 备案已提交并进入审核
- [ ] 数据权限矩阵可执行（不是纸面）
- [ ] 安全审计日志可查询
