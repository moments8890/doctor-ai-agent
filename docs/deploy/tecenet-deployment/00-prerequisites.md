# 00｜前置检查与项目准备

## 目标

在正式上云前，先把项目、人员、账号、交付口径统一，避免中途返工。

## 子清单

1. 项目范围确认
- [ ] 本周目标定义为“技术上线 + 小流量灰度 + 备案提交”
- [ ] 明确本周不承诺“备案最终办结”
- [ ] 确认上线范围：WeChat 文本/语音/图片/任务通知全链路

2. 环境与代码基线
- [ ] 创建发布分支：`release-cn-week1`
- [ ] 锁定依赖版本（`requirements.txt`、前端 lockfile）
- [ ] 执行一次完整测试：`bash scripts/test.sh all`
- [ ] 导出现有 SQLite 备份副本

3. 角色分工
- [ ] 云资源负责人（账号、网络、权限）
- [ ] 应用负责人（部署、配置、回滚）
- [ ] 数据负责人（DB迁移、校验）
- [ ] 运维负责人（监控、告警、值班）

4. 发布约定
- [ ] 每天固定一个发布窗口
- [ ] 约定回滚条件（错误率、时延、关键功能失败）
- [ ] 统一版本命名：`vYYYYMMDD-HHMM-gitsha`

5. 验证样本准备
- [ ] 准备 10 条文本样本（心内/肿瘤）
- [ ] 准备 5 条语音样本
- [ ] 准备 5 张图片样本（化验单/检查结果）
- [ ] 准备 3 条异常样本（空输入、噪声音频、模糊图片）

## 输出物

1. 本周上线目标说明
2. 团队分工表
3. 样本验证包

## 验收

- [ ] 团队对目标和边界达成一致
- [ ] 发布分支与回滚规则已生效
