# 04｜稳定在线与运维保障（高可用/监控/告警/备份）

## 目标

确保服务 7x24 小时稳定在线，具备故障可观测、可恢复、可追溯能力。

## 子清单

1. 高可用基础
- [ ] 至少双实例部署（避免单点）
- [ ] CLB 健康检查 + 自动摘除异常实例
- [ ] 后台任务处理从单进程改为独立 worker（避免重复和阻塞）

2. 可观测性
- [ ] 日志接入 CLS（应用日志、访问日志、错误日志）
- [ ] 指标接入腾讯云监控（CPU/内存/QPS/错误率/延迟）
- [ ] 链路追踪（可选，至少保留 request_id）

3. 告警策略
- [ ] 5xx 错误率阈值告警
- [ ] `/healthz` 连续失败告警
- [ ] DB 连接池耗尽告警
- [ ] 回调失败与任务通知失败告警

4. 数据可靠性
- [ ] TencentDB 自动备份开启（每日）
- [ ] 每周一次恢复演练（备份可用性验证）
- [ ] COS 跨区复制（可选）
- [ ] 关键业务表定期快照导出

5. 安全
- [ ] WAF 规则启用
- [ ] 仅允许 TLS1.2+
- [ ] 管理端口只对白名单开放
- [ ] 密钥轮换计划（至少季度）

6. 运维流程
- [ ] 故障分级：P0/P1/P2
- [ ] On-call 值班与升级路径
- [ ] 发布窗口与变更审批
- [ ] 事故复盘模板（RCA）

## 本步骤输出物

1. 监控面板链接
2. 告警规则清单
3. 备份恢复演练记录
4. 运维值班与故障响应手册

## 验收

- [ ] 关键告警可在 5 分钟内触发
- [ ] 故障恢复时间（RTO）达到团队设定目标
- [ ] 一次演练确认可从备份恢复业务数据
