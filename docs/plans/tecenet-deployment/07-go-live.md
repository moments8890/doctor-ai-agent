# 07｜上线日执行与回滚清单

## 目标

上线当天按固定顺序执行，确保可快速止损和恢复。

## 上线前 2 小时

1. 冻结与通知
- [ ] 暂停非上线相关代码合并
- [ ] 通知业务/运维/测试进入发布窗口

2. 备份与快照
- [ ] 执行数据库备份并记录时间点
- [ ] 导出当前生产配置（脱敏）
- [ ] 记录当前线上镜像 tag

3. 预检
- [ ] 发布流水线最近一次成功
- [ ] 依赖云服务健康（DB/COS/CLB/TCR/CLS）
- [ ] LLM 主备接口连通性检查通过

## 上线执行

1. 发布
- [ ] 发布新镜像到 TCR
- [ ] 执行滚动发布（或蓝绿切换）
- [ ] 观察健康检查连续 5 分钟稳定

2. 核心冒烟
- [ ] WeChat 文本消息闭环通过
- [ ] WeChat 语音消息闭环通过
- [ ] WeChat 图片消息闭环通过
- [ ] 任务通知发送与完成闭环通过

3. 监控确认
- [ ] 5xx 错误率低于阈值
- [ ] P95 延迟在阈值内
- [ ] 回调成功率达标

## 回滚条件（任一触发立即回滚）

1. 关键功能连续失败超过阈值
2. 5xx 错误率持续超阈值
3. 数据写入异常或出现严重一致性问题

## 回滚步骤

1. 切回上一稳定镜像 tag
2. 恢复上一版本配置
3. 再次执行核心冒烟
4. 发布故障通报并冻结后续发布

## 上线后 24 小时

1. 每 2 小时巡检一次核心指标
2. 汇总异常样本并归因（模型/代码/网络/数据）
3. 输出《上线日复盘》与次日修复计划
