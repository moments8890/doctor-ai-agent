# 提交审核指南

## 提交前检查清单

- [ ] 华为开发者账号已通过实名认证
- [ ] 已在 AppGallery Connect 创建应用
- [ ] 软著证书已获取
- [ ] 隐私政策页面已部署且可访问
- [ ] 签名 APK 已构建并在真机测试通过
- [ ] 商店图标（216x216）已准备
- [ ] 截图（≥3 张）已准备
- [ ] 应用描述已编写

## 提交步骤

### 1. 登录 AppGallery Connect

访问 https://developer.huawei.com/consumer/cn/service/josp/agc/index.html

### 2. 进入应用信息页

「我的应用」→ 选择「鲸鱼随行」→「应用信息」

### 3. 填写应用信息

#### 基本信息
- 应用名称：医生助手
- 应用分类：健康管理
- 内容分级：按提示完成分级问卷
- 隐私政策 URL：填入已部署的隐私政策链接

#### 版本信息
- 上传 APK 文件
- 填写版本说明（首次上架写「首次发布」即可）
- 支持的设备：手机

#### 素材上传
- 上传应用图标
- 上传截图
- 填写应用描述（简介 + 详细描述）

#### 版权信息
- 上传软著证书扫描件

### 4. 提交审核

确认所有信息无误后，点击「提交审核」。

## 审核周期

| 阶段 | 预计时间 |
|------|---------|
| 初审（机器） | 几小时 |
| 人工审核 | 1-3 个工作日 |
| 驳回后重新提交 | 1-2 个工作日 |

## 常见驳回原因及解决方案

### 1. 隐私政策不合规

**驳回原因**：隐私政策缺少必要条款或无法访问

**解决方案**：
- 确保隐私政策 URL 可正常访问
- 包含数据收集类型、用途、存储、共享、用户权利等必要条款
- 参考 `03-privacy-policy.md` 中的模板

### 2. 缺少权限说明

**驳回原因**：应用申请了权限但未说明用途

**解决方案**：
- 当前应用只申请了 `INTERNET` 权限，风险较低
- 如果后续添加了相机、存储等权限，需要在应用内首次使用时弹窗说明

### 3. 功能不完整

**驳回原因**：审核员发现功能无法正常使用

**解决方案**：
- 提供测试账号（在审核信息中填写）
- 确保 API 服务器正常运行
- 确保测试账号下有示例数据

### 4. 资质问题

**驳回原因**：健康类应用需要额外资质

**解决方案**：
- 如果被要求提供额外资质，可以调整应用描述
- 强调是「健康管理工具」而非「诊断设备」
- 在描述中明确声明「AI 内容仅供参考，不构成诊断依据」
- 如需资质可咨询当地卫健委

### 5. 截图与实际不符

**驳回原因**：商店截图与安装后的实际界面不一致

**解决方案**：
- 使用真实应用截图，不要 PS
- 可以添加文字说明边框，但核心内容必须是真实截图

## 审核通过后

1. 应用会在 1-2 小时内在华为应用市场上线
2. 可以在 AppGallery Connect 查看下载量、评分等数据
3. 后续更新：上传新 APK → 填写更新说明 → 提交审核

## 提供测试账号

在提交审核时，建议在「审核备注」中提供：

```
测试账号信息：
- 医生端登录：手机号 【测试手机号】
- 验证码/密码：【测试密码】

应用服务器地址：https://api.doctoragentai.cn
请确保审核期间服务器正常运行。
```

> **重要**：审核期间务必保证后端服务正常运行，否则审核员无法测试功能会直接驳回。
