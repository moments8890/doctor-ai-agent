# 鲸鱼随行智能医疗助手软件 V1.0

# 软件说明书

---

## 1. 引言

### 1.1 软件概述

鲸鱼随行智能医疗助手软件（以下简称"鲸鱼随行"）是一款面向医疗专业人员的智能化工作台应用。本软件通过人工智能技术辅助医生完成日常诊疗工作，包括患者管理、病历记录、任务跟踪、AI辅助分析等核心功能。

软件采用前后端分离架构，前端基于 React 框架开发，后端基于 Python FastAPI 框架，支持 Android 移动端和 Web 桌面端两种使用方式。

### 1.2 软件功能摘要

本软件主要包含以下功能模块：

| 序号 | 功能模块 | 功能描述 |
|------|---------|---------|
| 1 | 用户认证 | 医生和患者双角色登录与注册 |
| 2 | 首页简报 | 每日工作摘要、待处理任务提醒 |
| 3 | AI 对话助手 | 自然语言交互，辅助诊疗决策 |
| 4 | 患者管理 | 患者信息的创建、查询、编辑 |
| 5 | 病历管理 | 多类型病历的录入、查看、导出 |
| 6 | 任务管理 | 诊疗任务的创建、跟踪、完成 |
| 7 | 患者问诊 | AI 引导的患者预问诊流程 |
| 8 | 设置管理 | 医生信息、科室、知识库配置 |

### 1.3 运行环境

| 项目 | 要求 |
|------|------|
| 操作系统 | Android 6.0（API 23）及以上 |
| 设备类型 | Android 手机 |
| 网络要求 | 需要互联网连接 |
| 存储空间 | 约 10 MB |

### 1.4 技术特点

- **AI 驱动**：集成大语言模型，支持自然语言病历录入、智能诊断辅助
- **多端适配**：同时支持移动端和桌面端，自适应布局
- **中文优化**：界面语言和交互流程针对中国医疗场景设计
- **数据安全**：HTTPS 加密传输，医疗数据隔离存储

## 2. 安装说明

### 2.1 安装方式

用户可通过以下方式安装本软件：

1. 从华为应用市场搜索"鲸鱼随行"，点击"安装"按钮
2. 系统自动下载并安装应用（约 3.3 MB）
3. 安装完成后，点击"打开"即可使用

### 2.2 首次使用

1. 打开应用后进入登录页面
2. 新用户需先注册账号：
   - 医生用户：输入昵称、口令、邀请码完成注册
   - 患者用户：输入昵称、口令并选择所属医生完成注册
3. 已有账号的用户直接输入昵称和口令登录

## 3. 功能说明

### 3.1 用户认证模块

#### 3.1.1 登录

登录页面提供"医生"和"患者"两个标签页，用户选择对应角色后输入昵称和口令即可登录。

登录成功后：
- 医生用户进入医生工作台主界面
- 患者用户进入患者端界面

#### 3.1.2 注册

**医生注册流程：**
1. 在登录页面点击"注册新账号"
2. 输入昵称（姓名）
3. 设置口令
4. 输入邀请码（由管理员分发）
5. 点击"注册"完成

**患者注册流程：**
1. 切换到"患者"标签页
2. 点击"注册新账号"
3. 输入昵称
4. 设置口令
5. 从下拉列表中选择所属医生
6. 点击"注册"完成

#### 3.1.3 退出登录

在设置页面底部点击"退出登录"按钮，确认后清除本地会话信息并返回登录页面。

### 3.3 AI 对话助手模块

AI 对话助手是本软件的核心功能，医生可以通过自然语言与 AI 进行交互。

#### 3.3.1 对话界面

- 顶部显示当前工作上下文（当前患者、待处理草稿等）
- 中部为对话消息列表，包含医生发送的消息和 AI 回复
- 底部为输入区域，支持文字输入和语音输入

#### 3.3.2 快捷指令

提供以下快捷操作按钮：

| 指令 | 功能 |
|------|------|
| 今日摘要 | AI 自动生成当日工作摘要 |
| 新增病历 | 通过对话方式创建新病历 |
| 查询患者 | 通过自然语言搜索患者 |
| 诊断建议 | AI 提供诊断参考建议 |

#### 3.3.3 工作流程

1. 医生输入指令或自然语言描述
2. AI 理解意图并执行相应操作
3. 操作结果以结构化卡片或文本形式展示
4. 医生可继续追问或确认操作

### 3.4 患者管理模块

#### 3.4.1 患者列表

- 左侧面板显示患者列表，支持搜索和自然语言查询
- 按最近访问时间排序
- 每个患者条目显示：姓名、性别、年龄、病历数量

#### 3.4.2 新建患者

点击"新建患者"按钮，填写以下信息：
- 姓名（必填）
- 性别
- 出生年份
- 联系电话
- 身份证号

#### 3.4.3 患者详情

选中患者后，右侧面板显示患者详情：

**基本信息区域：**
- 患者姓名、性别、年龄
- 就诊统计（门诊次数、检验次数、影像次数）
- 联系电话、身份证号、建档日期

**操作按钮：**
- 添加标签：为患者添加分类标签
- 导出 PDF：导出患者资料为 PDF 文件
- 门诊报告：生成门诊报告
- 删除患者：删除患者档案（需确认）

#### 3.4.4 患者搜索

支持两种搜索方式：
- **关键词搜索**：输入患者姓名进行精确搜索
- **自然语言搜索**：输入描述性语句，AI 智能匹配

### 3.5 病历管理模块

#### 3.5.1 病历列表

在患者详情页的"病历记录"区域，显示该患者的所有病历。支持按类型筛选：

| 类型标签 | 包含的病历类型 |

本软件支持以下病历类型：

| 类型 | 说明 |

- 个人史
- 婚育史
- 家族史
- 体格检查
- 专科检查
- 辅助检查
- 初步诊断
- 治疗方案
- 医嘱及随访

#### 3.5.4 病历操作

- **查看病历**：点击病历卡片展开详细内容
- **编辑病历**：点击编辑按钮修改病历内容

### 3.7 患者问诊模块

#### 3.7.1 功能说明

患者问诊模块允许患者在就诊前通过 AI 引导完成预问诊流程。AI 会根据患者描述的症状，有针对性地询问相关病史和症状信息。

#### 3.7.2 问诊流程

1. 患者登录后进入问诊界面
2. AI 引导患者描述主要症状
3. AI 根据症状进行追问（如持续时间、伴随症状等）
4. 问诊完成后生成结构化的问诊总结
5. 问诊总结自动同步到医生端

#### 3.7.3 问诊总结

问诊完成后生成的总结包含：
- 主诉摘要
- 症状详情
- 相关病史
- 患者自述的用药情况

## 5. 技术特点

### 5.1 AI 能力

- **意图理解**：基于大语言模型的自然语言理解，识别医生操作意图
- **结构化输出**：将自由文本自动整理为结构化病历格式
- **上下文记忆**：在对话过程中保持上下文连贯性
- **多轮对话**：支持连续多轮交互完成复杂任务

### 5.2 安全机制

- **传输加密**：所有数据通过 HTTPS 加密传输
- **会话管理**：基于 Token 的会话认证，超时自动过期
- **权限隔离**：医生只能查看自己管理的患者数据
- **AI 免责**：所有 AI 生成的内容均标注"仅供参考"

### 5.3 性能优化

- **本地缓存**：用户偏好和界面状态本地持久化
- **按需加载**：仅加载当前可见的数据
- **响应式布局**：自动适配不同屏幕尺寸
