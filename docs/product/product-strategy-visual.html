统一定位

"这个AI想法跟我一样。" 医生的规则、病例、沟通风格塑造每一个AI输出。越用越懂你。

专科医生的AI随访副驾驶：管理复诊患者的诊断审核、随访回复、消息分诊。在医生已有的患者关系中运作（微信、门户、门诊渠道），不是独立的诊断/处方系统。

核心价值闭环

阶段AI做什么医生做什么
诊前结构化患者问诊 → 高质量病历审核、编辑病历
诊中鉴别诊断 + 检查 + 治疗建议，引用医生自己的规则确认、拒绝、编辑 — AI从编辑中学习
诊后用医生的语气起草随访回复，分诊患者消息，追踪任务审核草稿、一键发送，教AI新规则

医生为什么付费

1. AI说话像我

不是通用模板，是医生自己的风格和术语

2. 教完就变聪明

3. 省真实时间

每天省1小时+的消息处理和诊断审核时间

4. 规则透明可控

看得到AI用了哪条规则，随时增删改

核心差异化：医生数据主权

1. 数据完全归医生所有

患者资料、病历、对话、随访 — 所有权归医生。删除账号 = 数据彻底删除。

2. 知识可完全导入导出

专科Playbook、常见问答、回复模板 — 支持JSON/PDF/Markdown导出，零迁移壁垒。

3. 商业模式不靠数据锁定

我们卖的是工具，不是数据。医生用得好自然续费，不需要锁定。

医生工作台：4-Tab 导航

医生端 4 个底部 Tab，每个对应一类认知任务。设置合并进"我的AI"页面。

我的AI

  • AI身份卡（"张医生AI · 已学会12条规则"）
  • 7天引用数 / 待确认 / 今日处理
  • 快捷操作：预问诊码、待审核、处理随访
  • 我的方法：最近活跃的规则列表
  • CTA："继续教AI" / "导入病例"
→

患者

  • AI建议关注：基于医生规则触发
  • 每行显示AI知道/做了什么
  • "术后第7天 · 按你的规则 需复查CT"
  • 搜索 + 新建患者
→

审核

  • AI诊断建议队列
  • 引用透明："引用了你的规则" vs "未引用个人规则"
  • 确认 / 修改 / 排除 一键操作
  • 修改后提示"记成我的偏好？"
→

随访

  • AI用医生话术起草的回复草稿
  • 引用规则标签 + 一键发送
  • 即将到期的随访任务
  • 发送确认页（患者上下文 + 安全检查）
核心产品时刻： 医生打开随访Tab，看到AI已经用他的话术起草了回复并标注了引用规则，点"发送"。这就是每日习惯闭环。

诊前 → 诊中 → 诊后 全流程

阶段AI副驾驶医生
诊前患者扫码 → AI多轮问诊 → 结构化病历审核病历，补充关键问题
诊中鉴别诊断 + 检查 + 治疗方向，引用医生规则筛选确认，编辑 → AI学习偏好
诊后起草随访回复、分诊消息、追踪复查审核草稿、一键发送、教新规则
Day 1 价值： 如果第一周不能把医生的夜间消息处理时间砍掉至少1小时，这不是产品。

教学闭环 + 引用透明

"这个AI想法跟我一样"的核心机制：医生教 → AI学 → 引用透明 → 信任建立 → 医生继续教。

医生输入

  • 上传/粘贴诊疗规则
  • 编辑AI诊断建议
  • 修改AI随访草稿
  • 确认/拒绝推荐
→

AI学习

  • 编辑 → "记成我的偏好？"
  • 规则积累 → 引用频率追踪
  • 3次以上同类编辑 → 规则建议
  • 风格一致度指标（高/中/低）
→

引用透明

  • "引用了你的规则：术后头痛红旗"
  • "未引用个人规则" → 教AI一条
  • "AI按你的话术起草"
  • "按你的方法处理了 N 位患者"
引用是信任原语："引用了你的规则" vs "未引用个人规则" 的对比让医生想要教AI更多规则。缺失个性化变成了微转化时刻（"教AI一条"）。

知识三层架构

个人规则（Skills）

诊断规则、沟通话术、随访方案、危险信号

↓ 医生的"方法论" · AI每次输出都引用 · 可看/可改/可导入导出

病例记忆（RAG）

医生确认过的历史病例作为检索语料

↓ 新患者 → 找相似病例给LLM参考 · 确认越多匹配越准

患者上下文（Context）

这个患者的病历、用药、恢复阶段、对话历史

↓ 每次AI输出都带上下文 · 实现"了解这个患者"

"要有自己核心专科小模型在手才能安全" — 不是训练权重，而是积累个人规则+病例。这些才是真正属于医生的"小模型"，越用越强。

全功能持续使用

团队版（后期）

¥X/月/科室

多个医生 + 共享患者池

为什么能收这个价？ 1小时/天 × 30天 = 30小时/月的专家时间。神经外科医生的时间价值远高于¥3,000/月。实际是帮医生维护私域患者关系 = 维护收入来源。

不做的变现方式： 患者收费、药品提成、转诊佣金 — 利益冲突，破坏信任。
