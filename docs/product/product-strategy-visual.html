统一定位

为专科医生打造的AI助手：像医生助手一样问诊、写病历、给初步建议、做随访 — 医生只需做最终决策。

主任医师的真实工作模式：他让住院医/研究生先问好、写好病历、给初步建议，他来"查房"做决策。我们的AI就是这个角色。

AI助手做什么 vs. 医生做什么

AI助手

  • 问病史 — 语音/文字多轮采集，生成结构化病历
  • 初步诊断 — 基于病历+知识库+历史病例给出建议
  • 检查建议 — 推荐需要做的检查
  • 治疗建议 — 推荐治疗方向（药物类别，不含剂量）
  • 术后随访 — 回答常规问题，收集症状，提醒复查
  • 全程管理 — 按医生方案执行提醒和随访

医生（主任）

  • 审核病历 — 补充关键问题
  • 最终诊断 — 筛选、修改、做最终诊断
  • 确认检查 — 确认/调整检查方案
  • 确认治疗 — 确认/修改治疗方案
  • 处理异常 — 处理AI上报的异常和紧急情况
  • 制定方案 — 制定管理方案，调整计划

这不是： 诊所管理系统 / 通用医疗聊天机器人 / 取代医生 — AI只做初级工作，医生做所有决策。

核心差异化：医生数据主权

1. 数据完全归医生所有

患者资料、病历、对话、随访 — 所有权归医生。删除账号 = 数据彻底删除。

2. 知识可完全导入导出

专科Playbook、常见问答、回复模板 — 支持JSON/PDF/Markdown导出，零迁移壁垒。

3. 商业模式不靠数据锁定

我们卖的是工具，不是数据。医生用得好自然续费，不需要锁定。

三段式工作流

AI助手覆盖 诊前问诊 → 诊中辅助 → 诊后随访 全流程。

诊前（第一优先）

  • 患者扫码进入
  • AI语音/文字多轮问诊
  • 生成结构化病历
  • 患者确认 → 传给医生
"如果看到病人前就有一份好病例，他就会很舒服"
→

诊中（第二优先）

  • 结构化病历 + AI初步诊断
  • 鉴别诊断 + 检查建议
  • 治疗方向建议
  • 医生筛选确认 → 最终决策
"我在里面只要筛选，来决定下面的整治"
→

诊后（第三优先）

  • AI基于病历+医嘱自动回复
  • 收集症状，提醒复查
  • 医生制定管理方案
  • 异常上报 → 医生处理
"到复诊、用药调整时会提醒双方"
Day 1 价值： 如果第一周不能把医生的夜间消息处理时间砍掉至少1小时，这不是产品。

专科知识架构

不需要建传统知识图谱。三层架构让LLM拥有专科能力：

Skills（专科提示词知识）

神经科问诊话术、诊断规则、危险信号、治疗方案

↓ 告诉LLM"怎么问、怎么判断" · 医生可看/可改/可导入导出

RAG（病例历史匹配）

医生确认过的历史病例，embed后做相似匹配

↓ 新患者来了，找到相似病例给LLM参考 · 用得越多，匹配越准

Context（患者上下文）

这个患者的病历、用药、恢复阶段、对话历史

↓ 每次回复都带上，AI"了解"这个患者 · 医生可看/可修正

知识图谱（左手医生）我们的 Skills + RAG + Context
构建成本巨大（6000+疾病实体关系）低（markdown提示词+病例积累）
更新方式需要专人维护图谱医生确认病例自动积累
医生个性化不能天然实现（RAG匹配医生自己的病例）
专科深度广而浅深而专（按亚专科写skills）

"要有自己核心专科小模型在手才能安全" — 不是训练模型权重，而是积累专科知识库+病例库。这些才是真正属于医生的"小模型"。

商业模式

付费买家不是医院IT，是医生本人。

付费试用

¥8,000-20,000

2-3个月
历史对话导入、医生风格调优、专科playbook、白手套实施

续费（按月）

¥2,999-6,999/月

按活跃患者数或对话量分级
全功能持续使用

团队版（后期）

¥X/月/科室

多个医生 + 共享患者池

为什么能收这个价？ 1小时/天 × 30天 = 30小时/月的专家时间。神经外科医生的时间价值远高于¥3,000/月。实际是帮医生维护私域患者关系 = 维护收入来源。

不做的变现方式： 患者收费、药品提成、转诊佣金 — 利益冲突，破坏信任。
