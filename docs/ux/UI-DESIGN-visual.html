Design Philosophy

WeChat-native medical assistant. The app should feel like a professional extension of WeChat — familiar to Chinese smartphone users, not a foreign SaaS product.

1

Function over decoration

No visual element without purpose. If it doesn't help the doctor treat patients faster, remove it.

2

Flat and clean

No shadows, no gradients, no 3D effects. White cards on gray backgrounds, hairline borders.

3

Mobile-first

All doctor/patient views render as mobile layout. No responsive breakpoints. Desktop shows a phone frame.

4

Scan, don't read

Doctors have 30 seconds per patient. Brief labels for scanning, full text on expand.

5

Primary always right

Cancel LEFT, primary RIGHT. Color = intent (red=danger, green=go). WeChat/iOS/Alipay convention.

6

Consistent density

MUI 8px grid, 4px half-step. Only 0.5 increments (0.5, 1, 1.5, 2, 2.5, 3). Min 44px tap targets.

7

Chinese-first

All UI text in Chinese. Preserve medical abbreviations (CT, MRI, NIHSS).

8

Flat icons only

MUI outlined icons only. Never emoji, Unicode symbols, or icon fonts.

What We Don't Do

  • No skeleton screens for fast loads (<200ms) — show content directly
  • No emoji or Unicode symbol icons — use MUI outlined icons only
  • No toast notifications for expected outcomes — only for errors/async
  • No modal dialogs unless blocking is intentional
  • No hover effects — touch-first UI
  • No animations except page transitions and loading spinners
  • No color for color's sake — gray is the default, color = meaning

Platform & Frame

Desktop browsers render the app inside a phone-shaped container (MobileFrame). Ratio: 9:19.5. File: src/App.jsx

CSS min() for constraint-driven sizing. transform: translateZ(0) for containing block. On mobile (<520px): full screen, no frame.
