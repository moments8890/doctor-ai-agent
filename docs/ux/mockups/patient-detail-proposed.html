Proposed — 总览 / 病历 / 聊天

Tab structure

  • 总览 (shown): curated dashboard — AI summary, clinical context, 需要处理, recent records preview
  • 病历: flat chronological list of all clinical records, with type sub-filter (门诊/检验/影像/问诊)
  • 聊天 (N): full chat view inline; N = unread draft count

总览 tab sections (yellow tags A–C)

  • AAI 摘要: 1-2 sentence LLM summary of what's going on.
  • B临床资料: diagnosis / meds / allergies (red) / past history. Not demographics.
  • C需要你处理: pending review + draft replies. Tap → go to review page or chat tab.

Recent records intentionally omitted — they live on the 病历 tab one tap away. 总览 stays tight and skimmable.

Why this is better than the flat-list version

  • Answers the doctor's actual question first: "what's going on with this patient?"
  • Clinical context (meds, allergies) is visible in 2 seconds — matches what real EMRs prioritize
  • Flat list is still one tap away for deep-diving
  • Mirrors the Knowledge subpage pattern — consistent IA across the app
Cards use the rounded-group treatment (like Knowledge), not flush rows. That's appropriate here because each section has a different data shape.
