Proposed — card-per-rule

What changed

  • 1Card per rule — 12px grey gutter between. Each card is self-contained.
  • 2Confidence as a colored dot + 2-char label (高/中/低) — no "置信度：medium" chip eating space.
  • 3Category chip uses category color — 诊断 green, 用药 blue, 随访 orange, 通用 purple. Matches KnowledgeCard palette.
  • 4Evidence as a distinct block, grey bg, with a green "查看诊断 ›" chip on the right when there's a source link.
  • 5Action row: 排除 left (grey outline) · 采纳 right (green) — shorter labels, matches CLAUDE.md dialog convention.
  • 6Header bar with count + sort hint ("按置信度排序"). High-confidence items rise to top so doctor handles obvious accepts first.
  • 7Evidence link is a chip, not underlined text — matches app visual language.

Why cards (not flat list)

  • Each item is a decision, not a row to scan — cards force the eye to process one at a time
  • Per-card buttons feel anchored, not floating
  • Content is variable-length — cards absorb short/long rules gracefully
  • Matches the Knowledge subpage pattern (知识库 uses cards)

Optional (not in this pass)

  • Group cards by category with thin section labels
  • "全部采纳高置信度" bulk action at top — risky without user testing
  • Optimistic hide on 采纳 — card slides up + fades out before refetch
