# 医疗AI助手 — UX设计规格文档

## 概述

面向全科医生的微信风格移动应用，以AI对话为核心入口，帮助医生管理患者、任务和日常工作流程。

### 目标用户

全科医生（社区卫生服务中心、诊所），管理广泛患者群体和多种病症。

### 目标平台

- iOS
- Android
- 微信小程序

### 设计风格

微信经典扁平设计：
- 主题色：`#07C160`（微信绿）
- 背景色：`#ededed`
- 聊天气泡：白色（AI），`#95EC69`（用户）
- 头像：4px圆角方形
- 无阴影、无渐变、细线边框（0.5px `#e0e0e0`）
- 字体：PingFang SC / -apple-system（iOS），Noto Sans SC / Roboto（Android）

---

## 架构：Chat-First（对话优先）

AI对话是应用的核心枢纽。患者列表和任务页面是便捷视图，但所有操作都可以通过对话完成。AI会主动推送相关信息（如"今天有3位患者待复诊"）。

### 底部标签栏

| 标签 | 图标 | 功能 |
|------|------|------|
| AI助手 | 对话气泡 | 默认首页，AI对话交互 |
| 患者 | 双人图标 | 患者列表 + 详情 |
| 任务 | 任务图标 | 预约 + 待办事项 |
| 我的 | 人像图标 | 设置、文档、个人信息 |

**导航规则：**
- 标签栏始终可见，除非进入详情页（推入式导航）
- 当前标签：`#07C160` 绿色高亮
- 任务标签：红色badge显示待办数量
- 默认加载AI助手标签

## Screen 1: AI助手（默认首页）

### 布局

```
┌─────────────────────────┐
│       AI助手             │  ← 导航栏
├─────────────────────────┤
│                         │
│  [AI头像] 早安摘要气泡    │  ← AI主动推送今日摘要
│                         │
│        [用户气泡] [头像]  │  ← 用户消息（绿色）
│                         │
│  [AI头像] 回复气泡        │  ← AI回复（白色）
│                         │
├─────────────────────────┤
│  🎤  [输入消息...]  ⊕    │  ← 输入栏
├─────────────────────────┤
│ AI助手  患者  任务  我的   │  ← 标签栏
└─────────────────────────┘
```

### AI能力

1. **自然语言患者查询** — "查看陈梅最近的血糖报告"
2. **对话式新建患者** — "新建患者，张三，男，45岁，因胸痛来诊"
3. **对话式创建任务** — "下周二安排陈梅复诊"
4. **临床摘要** — "总结这位患者最近三次就诊"
5. **辅助决策** — "这些症状常见的鉴别诊断有哪些"

### 聊天气泡设计

- **AI消息**：白色背景，左侧绿色方形头像（"AI"），左箭头
- **用户消息**：`#95EC69`绿色背景，右侧灰色方形头像（医生姓氏），右箭头
- AI回复中可嵌入结构化卡片（患者信息、检验结果等），点击卡片 → 跳转对应患者详情页

### 输入栏

- 左：麦克风图标（语音输入）
- 中：文本输入框（"输入消息..."）
- 右：⊕ 圆圈加号 → 弹出操作面板（微信风格九宫格）：
  - 拍照：拍摄检验单/处方等，AI自动识别内容
  - 相册：从相册选择图片
  - 文档：上传PDF/Word文档
  - 患者档案：选择并分享已有患者的档案

### 语音输入

- 点击麦克风图标 → 输入框切换为"按住说话"按钮（微信风格）
- 按住录音，松手发送
- 语音自动转文字后发送给AI
- 再次点击麦克风图标 → 切回文字输入模式

### AI主动摘要

- 每日首次打开时触发，展示当日工作摘要
- 内容：待复诊患者数、紧急报告数、今日预约数
- 非首次打开不重复推送，但可通过"今日摘要"指令手动触发

### 布局

```
┌─────────────────────────┐
│         患者             │  ← 导航栏（居中标题）
├─────────────────────────┤

├─────────────────────────┤
│  [+] 新建患者            │  ← 第一项，虚线绿色边框
│      添加新的患者档案  >  │
├─────────────────────────┤
│  最近 · 6位患者          │  ← 分组标题
├─────────────────────────┤
│  [陈] 陈梅    今天09:30  │  ← 患者行
│  女,62岁·2型糖尿病·血糖危急 🔴│
├─────────────────────────┤
│  [李] 李伟    今天10:15  │
│  男,45岁·高脂血症·胆固醇偏高 🟠│
├─────────────────────────┤
│  ... 更多患者行 ...       │
└─────────────────────────┘
```

### 患者行设计

- **头像**：44x44px，灰色背景，显示患者姓氏（中文）
- **主行**：姓名（左），时间戳（右）
- **副行**：性别, 年龄 · 主要诊断 · 当前状态（单行省略）
- **状态指示**：红点 = 危急，橙点 = 需关注，无点 = 稳定

### 新建患者

- 列表第一项，虚线绿色边框 + 加号图标
- 点击 → 跳转AI助手标签，输入框预填"新建患者"，医生继续输入患者信息
- 对话是唯一创建入口，保持Chat-First一致性

### 新建患者流程（对话驱动）

**步骤1：对话输入**

医生在AI对话中输入（文字或语音），例如："新建患者 陈梅 女 62岁 糖尿病 对青霉素过敏"

**步骤2：AI提取 + 预览确认**

AI自动提取信息，在对话中生成预览卡片：

```
┌─────────────────────────┐
│  已为您提取以下信息：      │
│  ┌───────────────────┐  │
│  │ 姓名    陈梅       │  │
│  │ 性别    女         │  │
│  │ 年龄    62岁       │  │
│  │ 诊断    糖尿病     │  │
│  │ 过敏    青霉素     │  │
│  └───────────────────┘  │
│  [确认创建]  [编辑完善]   │
└─────────────────────────┘
```

- **确认创建** → 信息足够时直接保存（快速路径）
- **编辑完善** → 打开预填表单，补充更多信息

**步骤3：预填表单（如需编辑完善）**

```
┌─────────────────────────┐
│  < AI助手   新建患者      │
├─────────────────────────┤
│  基本信息（必填）         │
│  姓名          陈梅      │
│  性别       [男|女*|其他] │  ← 已预填
│  出生日期   1964年（推算）│  ← AI根据年龄推算
├─────────────────────────┤
│  联系方式                │
│  手机号          选填    │
│  身份证号        选填    │
├─────────────────────────┤
│  医疗信息                │
│  主要诊断     [糖尿病]   │  ← 已预填
│  过敏信息     [青霉素]   │  ← 已预填，红色标签
│  备注          选填      │
├─────────────────────────┤
│       [ 保存 ]           │
└─────────────────────────┘
```

- 表单分三组：基本信息（必填）/ 联系方式 / 医疗信息
- 必填字段：姓名、性别、出生日期
- 微信设置风格表单行（标签左，输入/选择右）
- 性别：分段选择器（男/女/其他）
- 出生日期：日期滚轮选择器
- 主要诊断：搜索/多选页面
- 保存成功 → 返回对话，AI确认"已创建患者陈梅"

### 搜索

- 顶部搜索栏，支持按姓名、病症、编号筛选
- 实时过滤

### 排序

- 按最近交互时间排序（最新在前）

## Screen 3: 患者详情

### 导航

- 推入式导航，隐藏底部标签栏
- 左上角返回按钮："< 患者"（绿色）

### 布局

```
┌─────────────────────────┐
│  < 患者    患者详情       │  ← 导航栏
├─────────────────────────┤
│  [陈] 陈梅               │
│  女·62岁·1962年5月12日    │  ← 基本信息卡
│  手机：138****6789  [危急]│
├─────────────────────────┤
│  2型糖尿病  高血压  骨质疏松│  ← 诊断标签
├─────────────────────────┤
│  📅 就诊记录      12次 > │
│  📋 处方记录    当前3种 > │  ← 菜单列表
│  🧪 检验报告    1项异常 > │    （微信设置风格）
│  ⚠️ 过敏信息     青霉素 > │
├─────────────────────────┤
│  最近就诊                │
│  2026年3月15日 · 门诊     │  ← 最近就诊摘要
│  主诉：头晕乏力一周...    │
├─────────────────────────┤
│  [AI对话咨询]  [新建任务]  │  ← 操作按钮
└─────────────────────────┘
```

### 基本信息卡

- 56x56px头像（姓氏），姓名、性别·年龄·出生日期、手机号
- 右上角状态标签：危急（红底）/ 需关注（橙底）/ 稳定（无标签）

### 诊断标签

- 灰色背景标签，水平排列，可换行

### 菜单列表（微信设置风格）

- 带彩色图标、标题、右侧摘要文字 + 箭头
- 就诊记录：绿色日历图标
- 处方记录：橙色文档图标
- 检验报告：蓝色试管图标（异常时红色文字）
- 过敏信息：红色警告图标

### 最近就诊

- 显示最近一次就诊的日期、类型和简要记录

### 操作按钮

按钮分两行排列：

- **第一行**（并排，次要按钮，白色带边框）：
  - **新建任务**：创建关联该患者的任务
  - **导出PDF**：弹出底部面板，选择导出内容（基本信息/诊断/就诊记录/处方/检验/过敏），自定义范围后生成PDF
- **第二行**（独占整行，主按钮，绿色大按钮）：
  - **AI对话咨询** → 跳转AI助手，自动发送"关于患者[姓名]"作为上下文，AI以该患者信息作答

### PDF导出

两个入口：
1. 患者详情页 → 点击"导出PDF" → 底部弹出内容选择面板
2. AI对话 → "导出陈梅的病历报告" → 弹出同样的选择面板（或AI根据指令自动选择内容）

内容选择面板：
- 基本信息（默认勾选，不可取消）
- 诊断信息、就诊记录、处方记录、检验报告、过敏信息（可勾选/取消）
- 就诊记录可选范围：最近5次 / 最近10次 / 全部
- 点击"生成PDF" → loading → 预览/分享/下载

### 子页面（就诊记录、处方、检验、过敏）

每个菜单项点击后推入对应列表页：

- **就诊记录**：时间倒序列表，每行显示日期 + 就诊类型 + 摘要，点击展开完整就诊记录
- **处方记录**：当前用药列表（药名、剂量、频次、开始日期）+ 历史处方
- **检验报告**：按日期分组，每项显示检验名称 + 结果 + 正常/异常标识，异常项红色高亮
- **过敏信息**：过敏原列表（药物、食物等）+ 反应描述，支持新增/编辑/删除

## Screen 4: 任务

### 分段控制

iOS风格分段器：**今天** | **待办** | **已完成**

- 今天：当日的预约和待办事项
- 待办：未来日期的任务
- 已完成：已完成和已取消的任务

### 布局（今天视图）

```
┌─────────────────────────┐
│         任务             │  ← 导航栏
├─────────────────────────┤
│  [今天]  待办   已完成    │  ← 分段控制
├─────────────────────────┤
│  预约 · 3项              │  ← 分组标题
├─────────────────────────┤
│  09:30  陈梅—复诊  [进行中]│  ← 预约行（时间突出）
│  10:15  李伟—初诊         │
│  14:00  刘静—产检         │
├─────────────────────────┤
│  待办事项 · 2项           │  ← 分组标题
├─────────────────────────┤
│  ○ 审核检验报告    紧急   │  ← 待办行（圆形勾选框）
│    陈梅—血糖异常          │
│  ○ 开具转诊单            │
│    赵明—转呼吸科          │
└─────────────────────────┘
```

### 预约设计

- 左侧突出显示时间（大字号，当前进行中的为绿色）
- 患者姓名 + 就诊类型
- 当前/下一个预约显示"进行中"绿色标签
- 简要描述

### 待办事项设计

- 圆形勾选框：红色边框 = 紧急，灰色边框 = 普通
- 点击勾选框 → 直接完成
- 每项关联患者姓名 + 简要描述

### 任务操作

三种交互方式：

1. **点击勾选框** → 快速完成
2. **左滑** → 显示"完成"（绿色）和"取消"（灰色）按钮
3. **点击任务行** → 进入任务详情页

### 任务详情页

- 推入式导航，显示完整任务信息
- 字段：类型、关联患者、创建时间、截止时间、状态
- 操作按钮：
  - **标记完成**（绿色主按钮）
  - **编辑**（白色按钮，修改描述/截止时间/优先级/关联患者）
  - **取消任务**（红色文字按钮）

### 已完成/已取消状态

- 已完成：绿色勾 + 删除线文字 + 完成时间
- 已取消：灰色叉 + 删除线文字 + 取消原因

## 全局设计规范

### 颜色系统

| 用途 | 颜色 |
|------|------|
| 主题色/激活态 | `#07C160` |
| 用户聊天气泡 | `#95EC69` |
| 危急/紧急 | `#FA5151` |
| 警告/需关注 | `#FF9500` |
| 正文黑 | `#111111` |
| 次要文字 | `#999999` |
| 细线边框（输入框等） | `#e0e0e0` |
| 列表分割线 | `#f0f0f0` |
| 页面背景 | `#ededed` |
| 卡片/列表背景 | `#ffffff` |
| 输入栏背景 | `#f5f5f5` |
| 标签栏背景 | `#f7f7f7` |

### 字号规范

| 用途 | 字号 |
|------|------|
| 导航标题 | 17px, weight 500 |
| 列表主文字 | 15px, weight 500 |
| 正文/聊天气泡 | 15px |
| 表单/菜单摘要 | 14px |
| 辅助信息 | 13px |
| 时间戳/标签 | 12px |
| 标签栏文字 | 10px |

### 间距规范

- 页面水平内边距：16px
- 列表行内边距：12px 16px
- 分组间距：8px
- 头像尺寸：44px（列表），56-60px（详情/资料）
- 头像圆角：4px

### 交互规范

- 列表行点击 → 推入式导航（隐藏标签栏）
- 返回 → 导航栏左侧绿色箭头 + 上级页面名称
- 左滑 → 显示操作按钮（完成/取消）
- 标签切换 → 无动画，即时切换
- 搜索 → 实时过滤

## 待细化（实现阶段）

以下内容在实现阶段详细定义：

- **登录/认证**：微信OAuth / 手机验证码 / 生物识别
- **空状态**：患者列表为空、任务为空时的引导页设计
- **加载/错误状态**：网络异常、AI无响应、数据加载失败的提示
- **数据隐私**：符合《个人信息保护法》和医疗数据管理要求
- **微信小程序适配**：导航差异、文件大小限制、API限制
- **横屏/平板适配**：仅支持竖屏，后续版本考虑平板
- **无障碍**：最小点击区域44pt、动态字体支持
