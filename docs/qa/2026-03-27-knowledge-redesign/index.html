Notes
NOTE Pre-existing Date display off by 1 day (timezone bug in formatDate)

Not caused by this redesign. The formatDate() function parses date strings like "2026-03-23" using new Date() which creates UTC midnight, but getDate() reads local timezone. In timezones behind UTC, dates display as 1 day earlier.

Example: item created_at "2026-03-23" displays as "3月22日" instead of "3月23日".

Fix: Use new Date(dateStr + "T00:00:00") to parse as local time, or split the string manually: const [,m,d] = dateStr.split("-"); return `${+m}月${+d}日`;
