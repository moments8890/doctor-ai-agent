## 一、全局观察

在查看具体失败案例之前，先看几个影响所有10个角色的全局问题。

### 观察1：AI助手从不主动提问

**现象：** 所有10个角色的对话中，AI助手发出的消息中**包含问号的数量为0**。AI只返回一个病历清单（✓/✗），从不主动向患者提问。

**示例（P5 陈大海，脑出血恢复期）：**
```
患者: 我是来复诊的，半年前脑出血，现在右侧肢体还不太好使。
AI助手: 收到，已记录。
  S: ✓ 主诉 ✓ 现病史 ✗ 既往史 ✗ 过敏史 ✗ 家族史 ✗ 个人史
  建议补充：既往史、过敏史、家族史、个人史
```

**问题：** AI说"建议补充"但**从不主动问**"您有什么过敏吗？"或"家里人有类似的病吗？"。这意味着标记为`volunteer=false`（需要被问到才会说）的信息**永远无法被收集到**。

**影响范围：** 这是P5、P6、P7、P8失败的根本原因。

**根本原因：** `interview_turn.py` 第220行附近，系统丢弃了LLM生成的自然语言回复，替换为服务端的病历清单。

**建议修复：** 保留LLM的自然语言回复（包含追问），病历清单作为附加元数据而非替代。

---

### 观察2：NHC规范评分对所有角色都是67/100

**现象：** 所有10个角色的NHC规范符合度评分都是67分（通过的和失败的都一样）。

**原因：** 主诉格式检查有3项：字数(✓)、包含病程(✓)、匹配可接受变体(✗)。第3项"变体匹配"使用的是**精确子串匹配**：

```python
variant_matched = any(v in cc_text or cc_text in v for v in cc_acceptable_variants)
```

比如P1的主诉是"体检发现右侧大脑中动脉M1段动脉瘤2周"，而可接受变体是"体检发现脑动脉瘤2周"——前者包含更多细节，不是后者的子串，所以匹配失败。

**影响：** 每个角色的NHC评分都被压低了约33分，综合评分被压低约10分。

**建议修复：** 用语义匹配（LLM判断）替代子串匹配，或者完全移除变体匹配检查（主诉的字数和病程检查已经足够）。

---

### 观察3：质量评审5位评审完全相同

**现象：** 9/10个角色的质量评分都是 `[8, 8, 8, 8, 8]`——5位评审给出完全相同的分数。

**原因：** 5位"评审"其实都是同一个模型（llama-3.1-8b-instant），温度=0.2。相同的输入+相同的模型+低温度 = 相同的输出。5位评审的设计初衷是通过模型多样性获得独立判断，但实际上等于1位评审。

**建议修复：** 用不同模型（如qwen3-32b + llama-3.1-8b混合），或提高温度到0.5-0.7增加变异。

### P2 李秀兰（脑卒中术后3个月复诊）— 综合评分83

**对话摘要（2轮）：**
```
AI: 您好！请描述您的症状。
患者: 我是来复诊的，3个月前做过脑梗取栓手术，现在恢复得还行，
      就是右手精细动作不太灵活。高血压8年了，高脂血症也有几年了，
      母亲也有高血压和中风。
AI: 收到，已记录。[病历清单]
患者: 没有药物过敏，不吸烟不喝酒...
```

**失败原因：** 2个关键事实缺失
- `pi_surgery`（关键）：期望"现病史"中包含取栓手术的详细信息（左侧颈内动脉闭塞，机械取栓术），但系统只在主诉中记录了"脑梗取栓术后3个月复诊"，现病史中缺少手术细节。
- `ph_stroke_surgery`（关键）：期望"既往史"中记录手术史，但既往史只有"高血压8年，高脂血症"，没有手术记录。

**分析：**
- **真实问题：** 系统把手术信息放在了主诉里，但没有同步到既往史和现病史。对于复诊患者，"3个月前的取栓手术"既是就诊原因（主诉），也是手术史（既往史），也是当前病程的起点（现病史）。系统只记录了一处。
- **评估框架问题：** 事实目录要求同一个手术事实出现在3个字段中（主诉、现病史、既往史），这对系统要求过高。如果系统在任一字段中捕获了手术信息，应该给予部分分数。

**建议修复：**
1. 系统端：当提取到"手术/取栓/支架"等关键词时，自动同步到 past_history
2. 评估端：接受"wrong_field"（信息在错误字段中）作为部分匹配
