### Usability Notes

- **Briefing "已完成" card** has no click target — the other 2 cards navigate somewhere, this one does nothing. Inconsistent.
- **Patient detail** on mobile requires scrolling to see records — profile section takes up significant space. The collapsible profile helps but defaults to expanded.
- **Chat send button bug** is confusing — user clicks the obvious green arrow, gets navigated away with no feedback.
- **No loading indicators** when chat message is being processed — user doesn't know AI is thinking.
- **"清空" (clear) in chat header** — no confirmation dialog. One accidental tap erases all context.

## Bugs Found

### ~~Critical~~ False Positive
1. ~~**Chat send button navigates away**~~ — Headless browser artifact. The `$B click @e13` command hit the wrong target (unlabeled button in accessibility tree). **Works correctly in real browser.** Enter key also works in headless.

### High
2. **Patient login shows wrong fields** — Patient tab shows 昵称 (nickname) + 口令 (passcode) instead of 手机号 (phone) + 出生年份 (YOB). May be dev-mode override, but should be verified.

### Medium
3. **"已完成" stat card has no destination** — 今日患者 → patients, 待办任务 → tasks, but 已完成 clicks do nothing visible.
4. **No "create task" from patient detail** — Doctor must navigate away from patient context to create a task.
5. **No loading/thinking indicator in chat** — After sending, no spinner or "AI is typing..." feedback.
6. **"清空" (clear chat) has no confirmation** — One-tap destructive action.

### Low
7. **Send button disabled state not obvious** — Button appears similar whether enabled or disabled. Green fill only appears when enabled, but the distinction is subtle.
