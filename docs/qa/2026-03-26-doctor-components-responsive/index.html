FieldReviewCard sacrifices content when width or text size increases

10:14 · Source audit · FieldReviewCard.jsx
  • The field value is forced onto one line with overflow: hidden, textOverflow: ellipsis, and whiteSpace: nowrap.
  • The two-button action group is fixed on the same row with flexShrink: 0, so larger text steals width from the actual medical content instead of wrapping below it.
  • The footer uses the same left/right button split, which gets tight once labels or accessibility text scaling increase.

InterviewCompleteDialog action row is too dense for small width and large text

10:18 · Source audit · InterviewCompleteDialog.jsx
  • The dialog keeps three actions on one horizontal row: 返回, 保存, and 保存并诊断 →.
  • The paper also adds side margins with mx: 3, reducing usable width further on 320px devices.
