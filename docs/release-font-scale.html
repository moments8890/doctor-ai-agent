Architecture

How It Works

User taps "大字" in Settings | v useFontScaleStore ──persist──> localStorage("doctor-font-scale") | v applyFontScale("large") ── sets module-level multiplier | v triggerFontScaleRerender() ── useSyncExternalStore version++ | v Root component re-renders ── recreates MUI theme | v TYPE Proxy returns scaled values ── 14px * 1.2 = 17px | v All components read TYPE.body.fontSize ── get 17 instead of 14

Key Design Decisions

  • Proxy on TYPE object — zero changes to 40+ components. They read TYPE.body.fontSize and get scaled values automatically.
  • 3 discrete levels, not a slider — simpler UX for non-tech doctors. Matches iOS/Android accessibility pattern.
  • Font only, not layout — padding/margins/icons unchanged. Prevents overflow on mobile viewports.
  • localStorage persistence — instant on page load, no backend round-trip needed.
  • useSyncExternalStore for re-render — lightweight, no React Context provider needed for existing components.

UX for Settings

Design for elderly users: No dialog popup. The 3 choices are always visible inline with a live preview sentence below. Tap once to switch — instant feedback, no save button needed.

Interaction Flow

  • Tap a size option → immediately scales all text on the current page
  • Navigate away → all other pages also display at the new scale
  • Close and reopen app → scale persists from localStorage
  • No network required → works offline

Risks & Mitigations

RiskMitigation
Text overflow at 1.35x in tight containers Only scales font, not layout. Chinese text wraps naturally. Selected 1.35x (not 1.5x) as max to stay within safe bounds.
Stale TYPE values in memoized components Root re-render via useSyncExternalStore propagates to all children. React.memo'd components with no prop changes are the only risk — none exist in this codebase.
Admin theme not scaled Intentional — admin is desktop-only, used by tech-savvy users. Not in scope.
