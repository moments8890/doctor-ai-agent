P0 B. style_guard.md contradicts followup_reply.md on urgent phrases

Both prompts get prepended into the followup-reply LLM call (FOLLOWUP_REPLY_LAYERS.style_guard=True). The LLM gets contradicting signals on every doctor reply.

FileSays
common/style_guard.md:44-50"立即去急诊"/"立即打120" are valid escalation forms
intent/followup_reply.md:61-83"严禁'立即去急诊'/'立即打120'/'马上去医院'等急诊指令"

Fix: keep the phrases in style_guard for surfaces that legitimately use them (diagnosis explains doctor's reasoning to doctor), but add a scoped override for followup_reply.

+ 本节适用于 **诊断/解释/医嘱解读** 类生成（diagnosis、daily_summary 等），
+ 用于把医生的处置意见传达给读者。
+
- 推荐做检查: `建议去医院做个[具体检查]` ...
- 紧急: `立即去急诊`（真实医生 5 次）
- 极紧急（罕见）: `立即打120`（真实医生 1 次，仅用于真实危及生命）

+ **意图覆盖**：在 `followup_reply` 流程（AI 起草给患者的随访消息）中，
+ 这一节作废 — followup_reply.md 规则 19 优先：AI 不替医生做紧急判断，
+ 遇风险信号一律转给医生而非输出急诊指令。

P0 C. Vision OCR doesn't strip <think> if model ignores /no_think

Live OCR pipeline at infra/llm/vision.py:104-126 returns raw text. If the model ignores the /no_think instruction at the top of vision-ocr.md, reasoning tags leak into the OCR output and downstream consumers see them as document content.

Fix: reuse agent.llm.clean_llm_output() at the wire, before returning.

     result = ...
-    return result
+    from agent.llm import clean_llm_output
+    return clean_llm_output(result)

Codex correction: do NOT also strip in vision_import.py/knowledge_ingest.py — the latter routes through llm_call which already cleans. Single source of truth at the wire.

Where I still disagree with Codex (~13% gap)

Stronger fix would be: drop those phrases from style_guard entirely AND add them only to the surfaces that need them (diagnosis.md). But that touches more files. Acceptable to ship the lighter version first and observe.
