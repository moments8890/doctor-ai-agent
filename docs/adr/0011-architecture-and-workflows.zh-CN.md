# ADR 0011 — 架构总览与核心工作流

本文档是
[ADR 0011](0011-thread-centric-conversation-runtime-and-deterministic-commits.md)
的配套文件，用于可视化 MVP 目标架构并追踪核心工作流在运行时中的完整路径。

## 架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                          通道适配器                                  │
│  ┌──────────┐   ┌──────────┐   ┌──────────┐                      │
│  │   Web     │   │  微信     │   │  语音    │                      │
│  └────┬─────┘   └────┬─────┘   └────┬─────┘                      │
│       │              │              │                              │
│       └──────────────┴──────────────┘                              │
│                      │                                             │
│            归一化输入 + 消息 ID                                     │
└──────────────────────┬─────────────────────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────────────────────┐
│                       会话运行时                                     │
│                                                                    │
│  ┌────────────────────────────────────────────────────────────┐    │
│  │  1. 去重检查（消息 ID）                                      │    │
│  │     重复？ → 返回之前的结果                                   │    │
│  └────────────────────────────┬───────────────────────────────┘    │
│                               │                                    │
│  ┌────────────────────────────▼───────────────────────────────┐    │
│  │  2. 加载 doctor_context                                     │    │
│  │     workflow: patient_id, pending_draft_id                  │    │
│  │     memory: candidate_patient, working_note, summary        │    │
│  └────────────────────────────┬───────────────────────────────┘    │
│                               │                                    │
│  ┌────────────────────────────▼───────────────────────────────┐    │
│  │  3. 草稿守卫                                                 │    │
│  │     pending_draft_id 已设置 + 确认？ → 提交，完成             │    │

│  │     pending_draft_id 已设置 + 其他？ → 重新提示，完成          │    │
│  └────────────────────────────┬───────────────────────────────┘    │

│  │     输出：{reply, memory_patch, action_request?}             │    │
│  └────────────────────────────┬───────────────────────────────┘    │

│  ┌────────────────────────────▼───────────────────────────────┐    │

│  │     执行：创建患者 / 绑定 / 创建草稿 / 等                     │    │
│  └────────────────────────────┬───────────────────────────────┘    │
│                               │                                    │

│                               │                                    │
│                           返回回复                                 │

┌─────────────────────────────────────────────────────────────────────┐
│                         持久化存储                                   │
│                                                                    │
```

### 上下文状态（每个医生）

```
doctor_context
├── workflow（确定性，代码所有）
│   ├── patient_id
│   ├── patient_name（显示缓存）
│   └── pending_draft_id
└── memory（临时性，LLM 可见）
    ├── candidate_patient
    ├── working_note
    └── summary
```

## 核心工作流

### 1. 首轮新建患者并记录病历（微信）

医生口述新患者病例，此前无上下文。

```
医生（微信）: "新建患者张三，男，45岁，主诉头痛三天"

  去重：新消息 ID → 继续处理
  加载上下文：空（无患者、无草稿）
  草稿守卫：无 pending_draft_id → 放行

  会话模型：
    输入：空上下文 + 单条轮次
    输出：{
      reply: "正在为张三建档并生成病历草稿...",
      memory_patch: {
        candidate_patient: {name: "张三", gender: "男", age: 45},
        working_note: "主诉头痛三天"
      },
      action_request: {type: "create_patient_and_draft", patient: {name: "张三", ...}}
    }

  提交引擎：
    1. 创建患者 "张三"
    2. 设置上下文 patient_id + patient_name
    3. 结构化步骤：原始轮次 → 格式化笔记
    4. 创建待确认草稿 → 设置 pending_draft_id

  持久化：保存上下文，追加轮次到 chat_archive

医生收到: "已为张三建档。病历草稿如下：
           【主诉】头痛三天...
           确认保存？"
```

### 2. 多轮笔记汇集与草稿确认

医生在多个轮次中提供临床细节，然后请求保存。

```
轮次 1: "说一下王芳的情况"
  草稿守卫：无待确认草稿 → 模型
  模型：{action: select_patient, patient_name: "王芳"}
  提交引擎：查找王芳 → 设置 patient_id
  回复："已选择患者王芳，请继续。"

轮次 2: "血压150/95，心率88"
  草稿守卫：无待确认草稿 → 模型
  模型：{memory_patch: {working_note: "BP 150/95, HR 88"}, action: none}
  回复："已记录。还有其他内容吗？"

轮次 3: "开了氨氯地平5mg，每日一次"
  草稿守卫：无待确认草稿 → 模型
  模型：{memory_patch: {working_note: "BP 150/95, HR 88; 氨氯地平5mg qd"}, action: none}
  回复："已记录处方。需要保存病历吗？"

轮次 4: "保存"
  草稿守卫：无待确认草稿 → 模型
  模型：{action: create_draft}
  提交引擎：
    结构化步骤：chat_archive 中的原始轮次 → 格式化笔记
    创建待确认草稿 → 设置 pending_draft_id
  回复："病历草稿：\n【...】\n确认保存？"

轮次 5: "确认"
  草稿守卫：pending_draft_id 已设置，输入 = 确认 → 处理
    提交引擎：确认草稿 → 正式病历
    清除 pending_draft_id
  回复："王芳的病历已保存。"
  （未调用模型）
```

### 3. 先记录内容，后提供患者姓名

医生口述临床内容但未指明患者。模型作为普通轮次处理澄清。

```
轮次 1: "记录一下，患者头晕两周，伴恶心呕吐"
  草稿守卫：无待确认草稿 → 模型
  模型：{
    action: clarify,
    reply: "请问这是哪位患者的记录？",
    memory_patch: {working_note: "头晕两周，伴恶心呕吐"}
  }
  提交引擎：action 为 clarify → 无持久化写入
  回复："请问这是哪位患者的记录？"

轮次 2: "赵六"
  草稿守卫：无待确认草稿 → 模型
  模型：{action: select_patient, patient_name: "赵六"}
  提交引擎：查找 "赵六" → 找到 → 设置 patient_id
  回复："已选择赵六。需要保存病历吗？"

轮次 3: "保存"
  草稿守卫：无待确认草稿 → 模型
  模型：{action: create_draft}
  提交引擎：
    结构化步骤：原始轮次（包括轮次 1 的内容） → 格式化笔记
    创建待确认草稿 → 设置 pending_draft_id
  回复："病历草稿：\n【主诉】头晕两周，伴恶心呕吐...\n确认保存？"

轮次 4: "确认"
  草稿守卫：pending_draft_id 已设置，输入 = 确认 → 处理
    确认草稿 → 正式病历
    清除 pending_draft_id
  回复："赵六的病历已保存。"
  （未调用模型）
```

### 4. 患者切换并发出警告

医生在 working_note 有内容但无草稿时切换患者。

```
状态：patient_id = 王芳，working_note = "BP 150/95, HR 88"，无 pending_draft_id

医生: "接下来看李明"
  草稿守卫：无待确认草稿 → 模型
  模型：{action: select_patient, patient_name: "李明"}
  提交引擎：
    当前患者为王芳，working_note 非空
    → 继续切换，清除上下文 + memory
    → 查找李明 → 设置 patient_id
  回复："注意：关于王芳的未保存记录已清除。已切换到李明。"
```

### 5. 待确认草稿阻止患者切换

医生在草稿等待确认时尝试切换患者。

```
状态：patient_id = 王芳，pending_draft_id = draft_123

医生: "看一下李明"
  草稿守卫：pending_draft_id 已设置，输入不是确认/放弃 → 重新提示
  回复："您有王芳的待确认草稿。请先确认保存或放弃草稿，然后再切换患者。"
  （未调用模型）
```

### 6. 重复投递（微信重试）

```
投递 1: MsgId = "wx_msg_abc123"
  → 去重检查：未见过 → 完整处理 → 回复已存储

投递 2（重试）: MsgId = "wx_msg_abc123"
  → 去重检查：已处理 → 返回之前的回复
  → 无模型调用、无副作用
```
