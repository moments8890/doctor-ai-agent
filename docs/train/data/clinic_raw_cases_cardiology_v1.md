# 心血管门诊原始病例语料库 v1（阶段：原始表达采集 — 极简单句）

## 语料说明

- 每条病例为**单句极简输入**，仅包含：患者姓名、性别、年龄、主诉/症状、偶有既往史
- **刻意不包含**：体格检查、辅助检查、诊断、治疗方案、随访计划
- 用途：测试 AI 在信息严重不完整时的**守纪律行为** —— 对无法从输入中推断的字段必须返回 `null`，不得猜测或编造
- 专科：心血管内科

## 字段预期行为（测试契约）

| 字段 | 预期 | 说明 |
|------|------|------|
| `chief_complaint` | **非空** | 可从输入直接提取 |
| `history_of_present_illness` | **非空** | 可从症状描述推断 |
| `past_medical_history` | 非空或 `null` | 仅在明确提及既往史时填写 |
| `physical_examination` | **`null`** | 输入中无体征数据 |
| `auxiliary_examinations` | **`null`** | 输入中无检查结果（Case 007 除外，提及心电图异常但无具体数值） |
| `diagnosis` | **`null`** | 不得凭症状猜测诊断 |
| `treatment_plan` | **`null`** | 不得凭症状推断用药或处置 |
| `follow_up_plan` | **`null`** | 无相关信息 |

> **重要**：diagnosis 和 treatment_plan 为 `null` 是**正确行为**，不是模型能力不足的表现。
> v2 语料（clinic_raw_cases_cardiology_v2.md）覆盖含完整临床内容的多风格输入。
