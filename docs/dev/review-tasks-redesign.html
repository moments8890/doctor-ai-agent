3. Task Notification System — Proposal
Notification Flow
How tasks surface to the doctor across the app — not just in the tasks tab.
Architecture
Task Sources
1. AI审核完成 → auto-create follow-up tasks
2. 医生手动 → "+ 新建" from tasks page or patient detail
3. 患者消息 → AI triage creates urgent tasks
4. 定时随访 → scheduled follow-up triggers
Where Tasks Surface
1. 任务 tab — full list, grouped by urgency
2. 患者详情 → "需要你处理" attention card
3. 我的AI → dashboard stat "待处理"
4. Bottom nav badge — red dot with count
5. Push notification (WeChat miniprogram)
Reminder System
• Each task has optional reminder time (default: due date 09:00)
• Reminders send WeChat template message to doctor
• Overdue tasks: escalate — red badge, top of list, daily 09:00 push
• Completed tasks auto-archive after 7 days
Data Model Addition
tasks table:
  + reminder_at: datetime
  + reminder_sent: boolean
  + source_type: enum (review, manual, triage, schedule)
  + source_record_id: integer (nullable)
  + notes: text
  + completed_at: datetime (nullable)
WeChat Push Notification
Template message sent to doctor's WeChat when task is due or overdue.
