设计说明（2026-04-20 修订：单决策 vs. 清单 + 紧凑样式）

  • 病例摘要 · 单卡多行：主诉 / 现病史 / 既往史 / 查体 合并到一张卡片里，label 左 60px 固定宽、value 右侧。与现有 RecordSummaryCard（ReviewPage.jsx:79）一致的视觉语法。
  • 诊断 · 1 条：单一判断——多个候选只增加噪音。按钮"采纳 / 修改"，无"换一条"。
  • 治疗方向 · 1 条：治疗方案是一条主线。按钮"采纳 / 修改"，无"换一条"。
  • 检查建议 · 单条展示 + 换一条循环：后端生成 2–4 条，UI 一次只露 1 条 AI 建议。采纳 → 该条收起为 ✓ 行，下一条自动出现；换一条 → 跳到下一条但不采纳；循环到最后时按钮变"已看完"。已采纳的 ✓ 行可叠加多条。
  • 紧凑间距：字段 margin 6px（不是 8px）、AI 行 padding 8/14/10（不是 10/14/12）、textarea min-height 38（不是 52）。整体垂直密度接近 antd-mobile List 的默认紧凑度。
  • Backend Phase 2 prompt trim：`differential` → 1；`treatment` → 1；`workup` → 2–4。
  • 采纳态：浅绿色单行+✓。"医生已在基础上修改"附注表示字段文本已偏离原 AI 文本。
  • 自定义按钮：+诊断 / +检查 / +治疗。诊断/治疗直接替换 AI 建议；检查追加一条。
  • 完成审核：显示未处理数；编辑中时按钮禁用。
  • 视觉 token：颜色/圆角/边线/字号全部对齐 theme.js APP/FONT/RADIUS。
