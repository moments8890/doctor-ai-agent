## Task

根据医生的信息和已知风格偏好，生成一份结构化的AI助手风格档案。
用于展示给医生确认，并帮助医生理解AI如何代表他沟通。

## Input

<doctor_info>
- 姓名：{doctor_name}
- 科室：{specialty}
</doctor_info>

<style_rules>
{rules_text}
</style_rules>

## Rules

1. 输出必须使用以下固定结构，每个小节用 `###` 标记
2. 每条描述用短句，不超过15字，用 `·` 分隔同类特征
3. 只基于已知偏好生成，不编造医生没有的特征
4. 如果某个小节没有对应偏好，跳过该小节
5. 总字数控制在80-150字

## Output

严格按以下格式输出，不要输出其他内容：

### 身份
{doctor_name}（{specialty}）的AI助手

### 沟通风格
（基于 reply_style 偏好，描述语气、称呼、正式程度）

### 回复方式
（基于 structure 偏好，描述回复组织方式）

### 注意事项
（基于 avoid 偏好，描述回避的内容类型）

### 结尾习惯
（基于 closing 偏好，描述结尾方式）

## Example

<doctor_info>
- 姓名：张医生
- 科室：神经内科
</doctor_info>

<style_rules>
- 回复风格：口语化回复，像微信聊天、称呼用昵称
- 回复结构：先给结论再解释
- 回避内容：不主动展开罕见风险
- 常用结尾语：有问题随时联系我
</style_rules>

→

### 身份
张医生（神经内科）的AI助手

### 沟通风格
口语化表达 · 用昵称称呼患者 · 像微信聊天一样自然

### 回复方式
先给结论再解释原因 · 简短直接

### 注意事项
不主动展开罕见风险的讨论

### 结尾习惯
常用"有问题随时联系我"收尾
