/no_think

## Task

从医生提供的回复示例中，提取体现其沟通风格偏好的规则。
目的：让医生通过粘贴"我希望AI像这样回复"的示例文本，快速建立个性化风格规则。

## Input

<example_reply>
{text}
</example_reply>

## Rules

### 提取原则
1. 只提取有明确文本证据的风格特征，不猜测。原因：过度推断会产生不准确的规则，降低医生对系统的信任
2. 最多提取 3 条最显著的特征。原因：少量高质量规则比大量模糊规则更有价值
3. 每条规则必须是可操作的指令（"结尾使用口语化叮嘱"），而非描述性观察（"医生的语气比较随和"）。原因：规则将直接注入AI生成prompt，必须是可执行的

### 字段定义
4. **reply_style** — 语气/称呼/正式程度。证据：使用"你"还是"您"，是否用敬语，语气严肃还是轻松，是否使用医学术语还是通俗语言
5. **closing** — 结尾用语/随访叮嘱模式。证据：有无结尾祝福语，随访叮嘱的具体形式，是否有"有问题随时联系"类表述
6. **structure** — 内容组织方式。证据：是否先给结论再解释，分段还是一段式，是否使用编号列表，段落长短
7. **avoid** — 明显缺失的常见内容类型。证据：回复中完全没有某类通常会出现的内容（如无注意事项、无情绪安抚、无用药提醒）。注意：仅当缺失的内容在该场景下通常会出现时才算"回避"
8. **edits** — 语言修辞习惯。证据：句子长短偏好，标点使用特点（如不用感叹号），中英文混用习惯，缩写使用

### confidence 定义
9. **high** — 特征在回复中多次出现或极其显著（如全文使用"你"而非"您"，每段都先给结论）
10. **medium** — 特征出现一次但足够明确（如结尾有明显的固定用语模式）
11. **low** — 特征不太确定，可能只是该场景下的自然表达而非偏好

### 规则文本要求
12. text 必须是祈使句格式，可直接作为AI指令使用（如"使用'你'而非'您'称呼患者"）
13. text 不得包含患者姓名、日期、具体病情等个人信息
14. text 长度控制在 10-30 字

## Output

输出 JSON，不要输出其他内容：

```
{{"rules": [{{"field": "字段名", "text": "一句话风格规则", "confidence": "low/medium/high"}}, ...]}}
```

- 所有 JSON key 使用英文，所有值使用中文或英文枚举值
- 不使用 null；无规则可提取时返回 `{{"rules": []}}`
- rules 数组最多 3 项

## Constraints

- 只输出 JSON，不要解释推理过程
- field 只能是 reply_style / closing / structure / avoid / edits 五选一
- confidence 只能是 low / medium / high 三选一
- 不提取医学事实内容（药物、剂量、诊断），只提取风格特征
- 不从回复的医学正确性角度评判，只关注沟通方式

## Examples

**示例1：口语化短回复风格**

<example_reply>
你好，片子看了，腰椎没大问题，退行性变，年纪大了都这样。不用太担心，注意别搬重东西，坐久了起来活动活动就行。有不舒服再来。
</example_reply>

→ {{"rules": [
  {{"field": "reply_style", "text": "使用口语化表达，语气轻松直接", "confidence": "high"}},
  {{"field": "structure", "text": "先给结论再简要解释，不分段不列表", "confidence": "high"}},
  {{"field": "closing", "text": "结尾用简短的随访叮嘱，如'有不舒服再来'", "confidence": "medium"}}
]}}
（全文口语化 → reply_style 高置信；结论前置且一段式 → structure 高置信；结尾叮嘱只出现一次但模式明确 → closing 中置信）

**示例2：正式详细型风格**

<example_reply>
您好，根据您本次的检查结果，情况如下：

1. 血常规：白细胞计数偏高（11.2×10⁹/L），提示可能存在感染
2. CRP：轻度升高（15mg/L），与感染表现一致
3. 胸片：右下肺野可见片状阴影

综合以上结果，考虑社区获得性肺炎可能性大。建议：
- 口服抗生素治疗5-7天
- 多饮水，注意休息
- 3天后复查血常规和CRP

如有发热加重或呼吸困难，请及时就诊。祝早日康复！
</example_reply>

→ {{"rules": [
  {{"field": "structure", "text": "使用编号列表逐项列出检查结果，分段组织内容", "confidence": "high"}},
  {{"field": "reply_style", "text": "使用'您'称呼患者，保持正式礼貌语气", "confidence": "high"}},
  {{"field": "closing", "text": "结尾包含紧急情况提醒和祝福语", "confidence": "medium"}}
]}}
（多处编号列表+分段 → structure 高置信；全文用"您" → reply_style 高置信；结尾有固定模式 → closing 中置信）

**示例3：极简风格（不含典型结构）**

<example_reply>
没事，继续吃药，下月复查。
</example_reply>

→ {{"rules": [
  {{"field": "reply_style", "text": "使用极简短回复，一句话概括结论和建议", "confidence": "medium"}},
  {{"field": "avoid", "text": "不写详细解释和注意事项", "confidence": "low"}}
]}}
（回复极短 → reply_style 中置信，样本太少不能确定是固定偏好；缺少解释和注意事项 → avoid 低置信，可能只是该场景不需要而非刻意回避）

**示例4：无明确风格特征的常规回复**

<example_reply>
您的血压控制良好，继续目前方案，3个月后复查。
</example_reply>

→ {{"rules": []}}
（回复过于普通，无法提取有意义的风格特征。避免从常规回复中过度推断。）
